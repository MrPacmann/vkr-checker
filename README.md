# Проверка оформления учебных документов

Локальное браузерное веб-приложение для предварительной автоматизированной проверки `.docx` и `.pdf` документов по требованиям оформления. Проект собирается как статический сайт на React, TypeScript и Vite, не требует backend и подходит для публикации на GitHub Pages.

## Что проверяет приложение

- структуру документа и обязательные разделы;
- заголовки, уровни и нумерацию;
- параметры страницы, поля, шрифт, интервалы, отступы и выравнивание;
- подписи и ссылки на рисунки, таблицы, формулы и приложения;
- список источников и ссылки на источники в квадратных скобках;
- часть типовых проблем оформления: лишние пробелы, пустые абзацы, повторы, некорректные подписи;
- формирование JSON- и HTML-отчёта с замечаниями и уровнем достоверности.

Автоматическая проверка не заменяет нормоконтроль, научного руководителя или ручную проверку документа. Она помогает заранее найти типовые ошибки и снизить количество правок.

## Конфиденциальность

Приложение работает локально в браузере:

- документы не отправляются на сервер;
- серверная часть отсутствует;
- облачные просмотрщики и внешние API для анализа документов не используются;
- сами документы не сохраняются в `localStorage`;
- в `localStorage` могут храниться только настройки интерфейса и профилей.

Не публикуйте в репозитории реальные курсовые работы, ВКР, отчёты, PDF-файлы, HTML-отчёты проверки и другие файлы с персональными или учебными данными.

## Локальный запуск

```bash
npm install
npm run dev
```

После запуска Vite откройте локальный адрес:

```text
http://127.0.0.1:5173/
```

Сайт работает, пока открыт терминал с `npm run dev`. Чтобы остановить локальный сайт, закройте терминал или нажмите `Ctrl+C`.

## Production-сборка

```bash
npm run typecheck
npm run build
```

Готовая статическая сборка создаётся в папке `dist/`.

Для локального просмотра production-сборки:

```bash
npm run preview
```
