# Проверка оформления учебных документов

Локальное браузерное веб-приложение для предварительной автоматизированной проверки `.docx` и `.pdf` документов по требованиям оформления. Проект собирается как статический сайт на React, TypeScript и Vite, не требует backend и подходит для публикации на GitHub Pages.

## Что проверяет приложение

- структуру документа и обязательные разделы;
- заголовки, уровни и нумерацию;
- параметры страницы, поля, шрифт, интервалы, отступы и выравнивание;
- подписи и ссылки на рисунки, таблицы, формулы и приложения;
- список источников и ссылки на источники в квадратных скобках;
- часть типовых проблем оформления: лишние пробелы, пустые абзацы, повторы, некорректные подписи;
- формирование JSON- и HTML-отчёта с замечаниями и уровнем достоверности.

Автоматическая проверка не заменяет нормоконтроль, научного руководителя или ручную проверку документа. Она помогает заранее найти типовые ошибки и снизить количество правок.

## Конфиденциальность

Приложение работает локально в браузере:

- документы не отправляются на сервер;
- серверная часть отсутствует;
- облачные просмотрщики и внешние API для анализа документов не используются;
- сами документы не сохраняются в `localStorage`;
- в `localStorage` могут храниться только настройки интерфейса и профилей.

Не публикуйте в репозитории реальные курсовые работы, ВКР, отчёты, PDF-файлы, HTML-отчёты проверки и другие файлы с персональными или учебными данными.

## Локальный запуск

```bash
npm install
npm run dev
```

После запуска Vite откройте локальный адрес:

```text
http://127.0.0.1:5173/
```

Сайт работает, пока открыт терминал с `npm run dev`. Чтобы остановить локальный сайт, закройте терминал или нажмите `Ctrl+C`.

## Production-сборка

```bash
npm run typecheck
npm run build
```

Готовая статическая сборка создаётся в папке `dist/`.

Для локального просмотра production-сборки:

```bash
npm run preview
```

## Публикация на GitHub Pages

Проект подготовлен к публикации в репозитории с именем `vkr-checker`. В `vite.config.ts` задан базовый путь:

```ts
base: "/vkr-checker/"
```

Ожидаемый публичный адрес после публикации:

```text
https://USERNAME.github.io/vkr-checker/
```

Замените `USERNAME` на имя владельца GitHub-аккаунта или организации.

Для публикации через GitHub Actions:

1. Создайте репозиторий `vkr-checker` на GitHub.
2. Загрузите проект в ветку `main`.
3. В настройках репозитория откройте `Settings -> Pages`.
4. В `Build and deployment` выберите `Source: GitHub Actions`.
5. Запустите workflow `Deploy to GitHub Pages` или сделайте push в `main`.

Workflow находится в `.github/workflows/deploy.yml`. Он устанавливает Node.js 20, выполняет `npm ci`, `npm run typecheck`, `npm run build` и публикует папку `dist` на GitHub Pages.

## Ограничения

- Браузер и Microsoft Word могут по-разному рассчитывать постраничную разметку.
- Не все параметры оформления явно доступны в DOCX: часть может наследоваться из темы, шаблона или системных шрифтов.
- PDF используется как дополнительный визуальный слой и не всегда даёт точную структурную информацию.
- OCR и визуальные эвристики не должны быть единственным основанием для критических ошибок.
- Итоговый процент соответствия является ориентировочным.

## Не добавлять в репозиторий

В репозиторий нельзя коммитить:

- реальные `.docx`, `.doc` и `.pdf` документы студентов;
- экспортированные `*-report.html`;
- папки `reports/`, `uploads/`, `test-documents/`, `student-works/`;
- файлы с персональными данными;
- локальные `.env` файлы;
- `node_modules/`, `dist/`, `build/` и другие артефакты сборки.

Эти правила добавлены в `.gitignore`, но перед публикацией всё равно проверьте содержимое коммита.
